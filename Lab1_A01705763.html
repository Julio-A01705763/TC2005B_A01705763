¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

Son el formato de comunicación entre el cliente y servidor web.

  • Get: Reaiza una peticion en específico
  • Head: Este método de petición es similar al método HTTP GET, sin embargo no retorna ningún contenido HTTP Response.
  • Post: Puede enviar datos al servidor por medio del cuerpo
  • Put: Es similar al método de petición POST, solo que el método PUT es idempotente; es decir puede ser ejecutado varias veces y tiene el mismo efecto
  • Patch: Este método se emplea para modificaciones parciales de un recurso en particular.
  • Delete: Este método de petición permite eliminar un recurso específico.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Se usaria el metodo post por que:

  • El método POST no tiene límite de cantidad de información a enviar.
  • La información proporcionada no es visible, por lo que se puede enviar información sensible.
  • Se puede usar para enviar texto normal así como datos binarios (archivos, imágenes...).

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

No es responsabilidad del desarrollador, debido a que las razones para que ocurra un error 404 no dependen de la pagina. Algunos ejemplos puden ser:

  • La dirección URL o sus contenidos se han eliminado o cambiado.
  • El enlace no se colocó correctamente.
  • El servidor web responsable no está en funcionamiento o hay problemas de conexión.
  • El nombre de dominio solicitado no puede ser convertido en una dirección IP por el Domain Name System.
  • El nombre de dominio (ya) no existe.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Si debido a que el error HTTP 500, indica que el servidor ha encontrado una condición inesperada que le impidió cumplir con la solicitud para ingresar al sitio web. Debido a esto es responsabilidad del desarrollador arreglar el problema que impide que los usuarios puedan acceder a su pagina.

Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

Es preferible utilizar los porcentajes, esto para que la pagina se pueda adaptar a los diferentes tamaños de pantalla que puedan tener los usuarios, ya que si usamos px o pt, solo lo estaremos acomodando con forme a nuestra pantalla, y no todos tendran una pantalla igual a la nuestra; aun que para algunos casos es necesario usar px y pt cuando queremos valores de trabajo especifico
